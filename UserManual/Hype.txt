
Conk is a versatile game construction set. You provide the graphics and some rules to
govern players and bad dudes. All the behind-the-scenes hair-tearing, caffeine
overdosing programming has already been taken care of.
Conk consists of three editors (paint program, background map editor and everything-else
editor) for creating game files, and a runtime program which plays the game.

Requires workbench 2.04+, hard drive and at least two megs of memory.

Runtime program requires any 1.3+ Amiga system.

Produce platformers, scrolly horizontal shoot-em-ups, Asteroids/StarDust style games,
Defender-style games, 'Shooting-gallery' games, top-down racing games and many other
styles.

Editors are system friendly and will run on 24bit boards with intuition support.
Runtime program requires an Amiga Chipset (and compatible monitor).

Hard drive installable - both Conk itself and the resulting games.

Full paint program included, or import IFF pictures/animbrushes from any source.

Comes with n fully playable games, plus assorted demos and clip art.

Play IFF anims for title screens/intro sequences.

Tracker music modules supported.

AGA support.

Games may have from 1 to 256 Colours.

Sprite support.

Multiplayer support.

IFF 8SVX and raw sound sample support.

Produces stand-alone games which may be freely distributed - you don't need the Conk
package to play them.

Multitasking - The runtime program takes over the display hardware, but leaves other
system resources intact. Play games while downloading... On networked machines...

Supports any configuration - make up a _serious_ game that requires an '060 and
20 megs of hard drive space...

Runtime program has intelligent file caching - keeps loading to a minimum.
